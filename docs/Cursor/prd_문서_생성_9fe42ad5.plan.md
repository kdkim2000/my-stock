# 주식 매매일지 대시보드 앱 PRD 문서 생성 계획

## 목표

사용자가 제시한 5개 섹션(프로젝트 개요, 대상 사용자, 핵심 기능, 데이터 스키마, UI/UX)을 반영한 **Product Requirements Document**를 [docs/PRD.md](docs/PRD.md)에 생성한다.

## 문서 구조 및 반영 내용

### 1. 생성할 파일

- **경로**: `docs/PRD.md`
- **전제**: `docs` 디렉터리가 없으면 파일 작성 시 경로를 함께 생성 (또는 선행으로 `docs` 폴더 생성)

### 2. PRD 문서 구성 (Markdown)


| 섹션                        | 내용                                                                                                               |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| **1. Project Overview**   | 주식 매매일지·포트폴리오 관리 앱 개요, 엑셀→구글 시트 붙여넣기 기반 동기화, KIS API 실시간 평가 손익 연동 요약                                             |
| **2. Target Audience**    | 개인 주식 투자자, 매매 복기/수익률 관리 목적, 엑셀·구글 시트 선호 사용자                                                                      |
| **3. Core Features**      | 3.1 Google Sheets 동기화(Read/Append), 3.2 KIS API 실시간 주가·평가손익, 3.3 인사이트 대시보드(지표 카드·차트), 3.4 매매 복기(상세 모달·저널/태그 에디터) |
| **4. Data Model**         | Google Sheets 컬럼: Date, Ticker, Type, Quantity, Price, Fee, Tax, Journal, Tags (예시 및 설명)                         |
| **5. UI/UX Requirements** | 반응형 웹(Mobile-first), shadcn/ui + Tailwind, 다크/라이트 모드                                                             |


### 3. 추가 권장 섹션 (선택)

- **6. Tech Stack / Out of Scope**: PRD 범위 명확화용 (기술 스택 참고만, 사용자 규칙의 칸반 스택과 별도로 이 앱의 예상 스택 명시 가능)
- **7. Success Metrics / Appendix**: 미정의 시 생략 가능

### 4. 작성 시 유의사항

- 모든 표는 마크다운 테이블 대신 **불릿 리스트**로 작성 (플랜 가이드: 표 사용 지양).
- 제목 레벨: `#` PRD 제목, `##` 대섹션, `###` 소섹션(3.1~3.4 등).
- 한글 문서, 용어 통일: 매매일지, 포트폴리오, 실현손익, 평가손익, 승률 등.

### 5. 작업 순서

1. `docs` 디렉터리 존재 여부 확인 후 없으면 생성 (또는 `Write`로 `docs/PRD.md` 저장 시 경로 자동 생성되는지 환경에 따라 처리).
2. 위 구조에 따라 [docs/PRD.md](docs/PRD.md) 단일 파일로 전체 PRD 본문 작성.

## 요구사항 매핑 요약

- **Database**: Google Sheets (Read + Append), 앱 내 저널링 → 시트 행 추가
- **External API**: 한국투자증권(KIS) Open API — 현재가, 평가 금액·평가 손익
- **Charts**: 6개월/1년 누적 수익금(꺾은선), 종목별 실현 수익률(막대), 매수/매도 비중·승률(파이)
- **지표 카드**: 총 실현손익, 현재 평가손익, 전체 승률(%), 총 자산
- **저널 UI**: 행 클릭 → 상세 모달, 매매 사유/감정/전략 태그 에디터

이 계획대로 진행하면 제공해주신 요구사항이 모두 반영된 PRD가 완성됩니다.
