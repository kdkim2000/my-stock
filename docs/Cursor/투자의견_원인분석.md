# 투자의견·증권사별 투자의견 미표시 원인 분석 (단계별)

## 1단계: UI에서 사용하는 데이터와 표시 조건

**위치**: `components/dashboard/TickerDetailContent.tsx` (약 517~595행)

- **투자의견** 섹션은 다음 두 블록으로 구성된다.
  - **종목 투자의견**: `opinionQuery.data?.tickerOpinion` (의견명, 목표가, 전망, 제시일)
  - **증권사별 투자의견**: `opinionQuery.data?.brokerOpinions` (테이블: 증권사, 의견, 목표가, 제시일)

**표시 분기**:
- `opinionQuery.isPending && !opinionQuery.data` → **"로딩 중…"**
- `opinionQuery.error`가 있음 → **"투자의견을 불러올 수 없습니다."**
- 그 외 → 데이터 영역 렌더
  - `tickerOpinion`이 없으면 종목 투자의견 블록 자체를 그리지 않음
  - `brokerOpinions`가 없거나 길이 0이면 **"증권사별 투자의견이 없습니다."**

**결론**:  
"가져오지 못한다"는 다음 둘 중 하나로 나타난다.

1. **에러 표시**: `opinionQuery.error`가 있어서 "투자의견을 불러올 수 없습니다."가 나오는 경우  
2. **빈 데이터 표시**: 에러는 없지만 `tickerOpinion`이 null이고 `brokerOpinions`가 빈 배열이라, 종목 의견은 안 나오고 "증권사별 투자의견이 없습니다."만 나오는 경우

## 2단계: opinionQuery 데이터 출처

**위치**: `TickerDetailContent.tsx` (약 55~64행)

- **useQuery** 사용
  - `queryKey`: `["kis", "opinion", code ?? "", revalidateTrigger]`
  - **code**: `stockInfoQuery.data?.code` (종목 6자리). stock-info가 로드된 뒤에만 값이 있음.
  - **enabled**: `!!code` → code가 없으면 요청 자체를 하지 않음.
- **queryFn**:  
  `GET /api/kis/opinion?code={code}&revalidate=1`(재갱신 시) 또는 `?code={code}`  
  - `res.ok === false`면 `throw new Error("Failed to fetch opinion")` → **opinionQuery.error** 설정됨.
  - 성공 시 `res.json()`을 반환 → `opinionQuery.data`는 `KisInvestmentOpinion` 형태.

**결론**:
- **에러가 나는 경우**: `/api/kis/opinion`이 4xx/5xx를 반환하거나, 네트워크 등으로 fetch가 실패한 경우.
- **에러 없이 빈 데이터인 경우**: `/api/kis/opinion`이 200으로 `{ tickerOpinion: null, brokerOpinions: [] }`를 반환한 경우.

## 3단계: API 라우트 동작

**위치**: `app/api/kis/opinion/route.ts`

- **GET** 요청만 처리. `code`가 6자리 숫자가 아니면 400.
- **fetcher**:  
  - `revalidate=1`이면 `getInvestmentOpinion(code)` 직접 호출  
  - 아니면 `unstable_cache(() => getInvestmentOpinion(code), ["kis-opinion", code], { revalidate: 600 })` 사용
- **성공 시**: `getInvestmentOpinion(code)`의 반환값을 그대로 JSON으로 200 응답.
- **예외 시**: `catch`에서 503 + `{ error: "Failed to fetch investment opinion" }` 반환 → 클라이언트에서 **opinionQuery.error** 발생.

**결론**:  
서버가 **throw**하는 경우에만 클라이언트가 "투자의견을 불러올 수 없습니다."(에러)를 본다.  
`getInvestmentOpinion`이 **예외를 던지지 않고** `{ tickerOpinion: null, brokerOpinions: [] }`만 반환하면 200이 나가므로, 클라이언트는 **에러 없이 빈 데이터**만 보게 된다.

## 4단계: getInvestmentOpinion에서 빈 값이 나오는 경우

**위치**: `lib/kis-api.ts` (약 615~675행)

반환 타입은 항상 `KisInvestmentOpinion`이며, **아래 조건일 때마다 `{ tickerOpinion: null, brokerOpinions: [] }`를 반환**하고(throw 없음) API는 200을 반환한다.

| 순서 | 조건 | 내용 |
|------|------|------|
| 1 | `code` 검증 실패 | 6자리 숫자가 아니면 즉시 empty 반환. (상세 페이지에서는 `stockInfoQuery.data?.code`를 쓰므로, 보통은 통과.) |
| 2 | **토큰 없음** | `getAccessToken()`이 `null` → empty. (KIS 인증 실패, 만료, 1분 1회 제한 등.) |
| 3 | **env 미설정** | `KIS_APP_KEY` 또는 `KIS_APP_SECRET`이 없으면 empty. |
| 4 | **HTTP 실패** | `fetch(url)`의 `res.ok === false` (4xx/5xx) → empty. |
| 5 | **KIS API 오류** | `body.rt_cd`가 존재하고 `"0"`이 아님 → empty. (KIS 쪽 에러 코드·메시지는 body에만 있고, 우리는 로그 없이 empty만 반환.) |
| 6 | **응답 배열이 아님/비어 있음** | `body.output` / `body.output2` / `body` 중 배열인 것을 쓰는데, 배열이 아니거나 `list.length === 0`이면 empty. |

**배열 추출 로직**:
```ts
const arr = body.output ?? body.output2 ?? body;
const list = Array.isArray(arr) ? arr : Array.isArray(body?.output2) ? body.output2 : [];
```
- KIS가 **output**에 단일 객체만 주고 **output2**에 목록을 주는 경우: 위 로직으로 `output2` 사용 가능.
- **output**이 배열이 아닌 객체(예: 요약 정보)이고 **output2**가 없거나 배열이 아니면: `list = []`가 되어 empty 반환.

**결론**:  
투자의견/증권사별 투자의견을 "가져오지 못하는" 대부분의 경우는,  
**2(토큰)·3(env)·4(HTTP)·5(KIS 오류)·6(응답 구조/빈 배열)** 중 하나에서 empty가 반환되기 때문이다.

## 5단계: 에러 메시지가 나오는 경우 (클라이언트에서 "불러올 수 없습니다")

- **API 라우트가 예외를 던지는 경우**만 503이 나가고, 그러면 클라이언트에서 `opinionQuery.error`가 설정되어 "투자의견을 불러올 수 없습니다."가 표시된다.
- `getInvestmentOpinion` **내부**는 위 표의 모든 경우에 **throw 없이** empty만 반환하므로,  
  현재 코드에서는 **라우트 쪽에서만** 예외가 날 수 있다.
  - 예: `unstable_cache` 또는 `getInvestmentOpinion` 내부의 `fetch`/`res.json()` 등에서 예기치 않은 예외가 발생한 경우.

## 6단계: 원인 정리 및 확인 방법

### 가능 원인 요약

| 현상 | 가능 원인 |
|------|------------|
| **"투자의견을 불러올 수 없습니다."** (에러) | `/api/kis/opinion` 503 또는 fetch 실패. 토큰/네트워크/서버 예외. |
| **로딩만 계속** | `code`가 비어 있어 `enabled: false` 상태이거나, 요청이 끝나지 않음. (stock-info 지연 등.) |
| **에러 없이 "증권사별 투자의견이 없습니다." + 종목 의견 없음** | `getInvestmentOpinion`이 empty를 반환한 경우. ① 토큰 null ② KIS_APP_KEY/SECRET 미설정 ③ KIS API 4xx/5xx ④ KIS body.rt_cd ≠ "0" ⑤ KIS가 목록을 배열로 주지 않거나 빈 배열 반환. |

### 단계별로 확인할 것

1. **환경 변수**  
   - `KIS_APP_KEY`, `KIS_APP_SECRET` (및 필요 시 `KIS_APP_URL`)이 설정되어 있는지.

2. **다른 KIS API 동작 여부**  
   - 같은 토큰을 쓰는 API(예: 종목 시세, stock-info)가 상세 페이지에서 정상 동작하면, 토큰·env 문제 가능성은 낮음.  
   - 그 API들도 안 되면 토큰/인증/네트워크를 먼저 의심.

3. **/api/kis/opinion 직접 호출**  
   - 브라우저 또는 curl: `GET /api/kis/opinion?code=005930`  
   - 200이면서 `{ tickerOpinion: null, brokerOpinions: [] }` → 위 ①~⑥ 중 하나(대표적으로 ⑤·⑥).  
   - 503 → 라우트에서 예외 발생.

4. **서버 로그**  
   - `[KIS] opinion error:` 가 찍히면 라우트 catch에서 예외가 난 것.  
   - `getInvestmentOpinion`은 실패 시 로그를 남기지 않으므로, **KIS 응답 구조/rt_cd 확인**을 위해 `body`를 개발 환경에서 한 번 로깅해 보는 것이 좋음.

5. **KIS 응답 구조**  
   - invest-opinion API가 **output / output2**에 어떤 형태로 목록을 주는지(배열인지, 단일 객체인지, 필드명 mbcr_name / stck_opnn_txt 등) 확인.  
   - 배열이 다른 키에 있거나, 1단계 래핑이 있으면 현재 `arr`/`list` 추출 로직이 빈 배열을 만들 수 있음.

## 요약

| 단계 | 확인 내용 | 결과 |
|------|-----------|------|
| 1 | 투자의견 UI가 사용하는 데이터 | `opinionQuery.data` (tickerOpinion, brokerOpinions). 에러 시 "불러올 수 없습니다.", 빈 데이터 시 "증권사별 투자의견이 없습니다." 등 |

| 4 | empty 반환 조건 | 토큰 null, env 미설정, res.ok false, rt_cd≠"0", output/output2 비배열 또는 length 0 |
| **원인** | | **① KIS 인증/토큰 실패 ② KIS_APP_KEY/SECRET 미설정 ③ KIS API 오류(rt_cd) 또는 4xx/5xx ④ KIS 응답이 배열이 아니거나 빈 배열** 등으로 empty가 반환되고, 이때 200이 나가면 UI는 에러 없이 빈 상태로 표시됨. |

## 적용된 수정 사항 (원인 파악 및 보강)

- **개발 로그 추가** (`lib/kis-api.ts`): `getInvestmentOpinion`에서 empty를 반환하는 단계마다 `[KIS] opinion: ...` 로그 출력 (NODE_ENV=development일 때).
  - `invalid code` / `no access token` / `KIS_APP_KEY or KIS_APP_SECRET not set` / `HTTP %d` / `body parse failed` / `rt_cd=%s msg1=%s` / `empty list (output/output2) code=%s keys=%s`
- **배열 추출 보강**: `extractListFromKisBody` 도입. `body.output` / `body.output2`가 (1) 배열이면 그대로 사용, (2) 객체이면 그 안의 배열을 찾거나, (3) 객체의 값들이 모두 객체이면 그 값 배열을 목록으로 사용 (예: `{ "1": {...}, "2": {...} }` 형태).
- 위 수정으로 서버 터미널에서 **어느 단계에서 empty가 반환되는지**와 **KIS 응답 body의 키**를 확인할 수 있으며, 응답 구조가 배열이 아닌 경우에도 목록을 추출할 수 있다.
