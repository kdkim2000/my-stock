# 네비게이션·메인·상세 전체 재구성 계획

## 1. 메인 페이지 정리

**현재**: [app/page.tsx](app/page.tsx) — 제목 + "대시보드로 이동" 링크만 있어 썰렁함.

**방안 (택 1)**  

- **A안 (권장)**: `/` 접속 시 즉시 `/dashboard`로 리다이렉트. `app/page.tsx`에서 `redirect('/dashboard')` 호출 또는 `next.config.mjs`에 `redirects` 설정.  
- **B안**: 메인을 랜딩으로 활용 — 요약 카드 1~2개 + "대시보드 보기" CTA 등 의미 있는 한 화면 구성.

계획 기본값: **A안(리다이렉트)** 으로 단순화.

## 2. 네비게이션·디자인 컨센 초기화 및 단순화

**대상**  

- [components/AppNav.tsx](components/AppNav.tsx): 현재 대시보드 + 종목별 분석/매매일지/전략별 성과 해시 링크 4개.  
- [components/dashboard/DashboardNav.tsx](components/dashboard/DashboardNav.tsx): 현재 대시보드에서 사용되지 않음(Grep 기준). 제거 또는 통합 검토.

**제안**  

- **AppNav**: "대시보드" 하나만 노출 (클릭 시 `/dashboard`). 상세 페이지일 때만 "← 대시보드" 백링크 유지. 해시 링크(종목별 분석, 매매일지 등) 제거하여 상단을 심플하게.  
- **DashboardNav**: 사용처 없으면 삭제.  
- **디자인**: [app/globals.css](app/globals.css)의 기존 calm 팔레트·profit/loss 시맨틱 유지. 추가 컨벤션은 "최소 변경" — 카드·테두리·간격은 현재 스타일 유지, 섹션 제목에만 아이콘 추가로 시각적 일관성 확보.

## 4. 상세 페이지 — 꼭 필요한 정보 + 표/차트 아이콘으로 시각화

**파일**: [components/dashboard/TickerDetailContent.tsx](components/dashboard/TickerDetailContent.tsx)

**4.1 아이콘 도입**  

- `components.json`에 `iconLibrary: "lucide"` 있으나 패키지 미설치. **lucide-react** 추가.  
- 섹션 제목에 아이콘 배치:  
  - 표 형태: `Table2` 또는 `LayoutList`  
  - 차트/그래프: `BarChart2` / `LineChart`  
  - 시세·가치: `TrendingUp` / `CircleDollarSign`  
  - 공시·참고: `FileText` / `Info`  
  - 매매일지: `Calendar` / `ClipboardList`  
  - 포트폴리오: `PieChart` 또는 `Wallet`  
  - 보조지표: `Activity`  
  (실제 매핑은 한글 제목과 1:1로 정리해 적용)

**4.2 한눈에 보이게 정리**  

- 상단: 종목명·현재가·전일대비·보유 요약 — 유지하되, 카드형 레이아웃 유지.  
- **시세 요약 / 가치평가**: 그리드 카드 유지, 섹션 제목에 아이콘만 추가.  
- **재무·비율·추정실적·매매동향·투자의견**: 테이블/카드 섹션에 `Table2` 또는 `BarChart2` 등 아이콘 추가.  
- **DART 매출·영업이익·당기순이익**: 이미 BarChart — 제목에 `BarChart2` 아이콘.  
- **현금흐름**: 테이블 — `Table2` 아이콘.  
- **공시**: `FileText`. **내 포트폴리오**: `Wallet` 또는 `PieChart`. **보조지표**: `Activity`. **참고 지표**: `Info`. **최근 매매 일지**: `Calendar` 또는 `ClipboardList`.  
- 테이블은 `thead`/`tbody` 구분·`bg-muted/40` 등 기존 스타일 유지해 가독성 확보.  
- "꼭 필요한 정보만": 섹션 삭제보다는 **섹션 제목 + 아이콘**으로 구분만 명확히 하고, 추후 필요 시 비율/추정실적 등을 접이식(accordion)으로 묶는 확장만 고려.

**4.3 (선택) 상단 목차**  

- 상세 페이지 상단에 "이 페이지 내 섹션" 앵커 링크 나열(아이콘+제목). 스크롤이 길 때 한눈에 파악 가능.
