# 공시 요약 미표시 원인 분석 (단계별)

## 1단계: UI에서 사용하는 데이터

**위치**: `components/dashboard/TickerDetailContent.tsx` (약 471~514행)

- "공시 요약" 섹션은 **사업의 내용**, **이사의 경영진단 (MD&A)**, **주석** 세 가지를 토글로 표시한다.
- 표시 텍스트는 모두 **`fundamentalData.document`** 에서 온다.
  - `fundamentalData.document?.businessOverview` → 사업의 내용
  - `fundamentalData.document?.mda` → 이사의 경영진단 (MD&A)
  - `fundamentalData.document?.notes` → 주석
- 값이 없으면 `"데이터를 불러오지 못했습니다. (document.xml 연동 후 표시)"` 가 나온다.

**결론**: 공시 요약이 안 나오는 이유는 `document` 의 `businessOverview`, `mda`, `notes` 가 비어 있기 때문이다.

---

## 2단계: document 는 어디서 오는가

**위치**: `hooks/useFundamentalData.ts`

- `useFundamentalData(code)` 가 **`/api/fundamental/financials?code=...`** 를 호출한다.
- 응답에서 `financialsQuery.data?.document` 를 꺼내 `document` 로 반환한다.
- 즉 **공시 요약 = API 응답의 `document` 필드**이다.

**결론**: `GET /api/fundamental/financials` 가 반환하는 `document` 내용이 비어 있으면 공시 요약이 표시되지 않는다.

---

## 3단계: API에서 document 를 어떻게 채우는가

**위치**: `app/api/fundamental/financials/route.ts`

```ts
const financials = await getFundamentalFinancials(code);
const document = financials?.corpCode
  ? await getDartDocumentSections(financials.corpCode)
  : {};
// ...
const body = { code, financials, document };
```

- `getFundamentalFinancials(code)` 로 재무 데이터와 `corpCode` 를 구한 뒤,
- **`getDartDocumentSections(corpCode)`** 를 호출해 `document` 를 만든다.
- 이 `document` 가 그대로 응답에 실려 프론트의 `fundamentalData.document` 가 된다.

**결론**: `document` 가 비어 있는지는 전적으로 **`getDartDocumentSections`** 구현에 달려 있다.

---

## 4단계: getDartDocumentSections 실제 구현

**위치**: `lib/dart-fundamental.ts` (약 167~202행)

```ts
export async function getDartDocumentSections(
  corpCode: string,
  _rceptNo?: string
): Promise<DartDocumentSections> {
  const key = process.env.DART_API_KEY?.trim();
  if (!key) return {};

  try {
    // TODO: 1) list API로 해당 corp_code의 최근 사업보고서 rcept_no 조회
    // TODO: 2) document.xml?rcept_no=... 호출하여 XML 스트림 수신
    // TODO: 3) '사업의 내용', '이사의 경영진단', '주석' 등 섹션 파싱
    // 현재는 뼈대만 반환
    const _url = `${DART_DOCUMENT_BASE}/document.xml?crtfc_key=${key}&rcept_no=${_rceptNo ?? ""}`;
    return {
      businessOverview: undefined,
      mda: undefined,
      notes: undefined,
    };
  } catch {
    return {};
  }
}
```

- **실제로 DART API를 호출하거나 XML을 파싱하는 로직이 없다.**
- 항상 `{ businessOverview: undefined, mda: undefined, notes: undefined }` 를 반환한다.
- 주석에도 "현재는 뼈대만 반환" 이라고 되어 있다.

**결론**: **공시 요약이 안 나오는 직접 원인은 `getDartDocumentSections` 가 아직 구현되지 않은 스텁(뼈대)이기 때문이다.**

## 5단계: 구현하려면 필요한 것 (참고)

DART에서 공시 문서 본문을 가져오려면 대략 다음 단계가 필요하다.

1. **목록 API**  
   - `https://opendart.fss.or.kr/api/list.json`  
   - `corp_code`, `pblntf_detail_ty=A001`(사업보고서) 등으로 **최근 사업보고서의 `rcept_no`(접수번호)** 조회

2. **문서 API**  
   - `https://opendart.fss.or.kr/api/document.xml?crtfc_key={key}&rcept_no={rcept_no}`  
   - 응답은 **ZIP 파일**이며, 그 안에 `document.xml` 이 들어 있음

3. **XML 파싱**  
   - ZIP 해제 후 `document.xml` 읽기 (인코딩: EUC-KR 등 처리 필요)  
   - XML 구조에서 **"사업의 내용"**, **"이사의 경영진단"**(MD&A), **"주석"** 해당 구간을 찾아 텍스트 추출

4. **반환**  
   - 추출한 텍스트를 `DartDocumentSections` 의 `businessOverview`, `mda`, `notes` 에 넣어 반환

현재는 1~3이 전부 TODO 상태이므로, **공시 요약을 실제로 채우려면 `getDartDocumentSections` 를 위 흐름대로 구현**해야 한다.
