# 로그 기반 데이터 미수집 원인 분석

분석 대상 로그: 삼성에스디에스(018260), LG전자(066570) 등 상세 페이지 조회 시 발생한 메시지

---

## 0. search-info "없는 서비스 코드" (PER/PBR/EPS 미표시)

### 로그
```
[KIS] search-info: rt_cd=1 msg1=없는 서비스 코드 입니다 code=066570
```

### 원인
- **위치**: `lib/kis-api.ts` — `getKisStockFundamentals(code)` 호출 시 사용하는 **종목정보(search-info)** API. TR ID: `FHKST02010100`(실전) / `VFHKST02010100`(모의).
- **의미**: KIS API에서 반환하는 **"없는 서비스 코드 입니다"**의 "서비스 코드"는 **종목코드(066570)가 아니라, 우리가 호출하는 API 서비스(TR ID)** 를 가리키는 경우가 많음.
- 즉, **해당 TR(FHKST02010100)이 사용 중인 앱키/계정에 등록되지 않았거나, KIS에서 해당 서비스를 제공하지 않는 상태**일 때 이 메시지가 나옴.

### 대응
| 조치 | 내용 |
|------|------|
| KIS 개발자포털 확인 | [한국투자증권 오픈API]에서 국내주식 **종목정보** 또는 **search-info**용 TR ID가 FHKST02010100이 맞는지, 해당 앱에 권한이 있는지 확인. |
| 환경변수 | `KIS_SEARCH_INFO_TR_ID`로 TR ID를 오버라이드할 수 있으면, KIS 문서에 안내된 최신 TR로 변경 후 재시도. |
| 화면 | search-info·opinion 모두 실패 시 PER/PBR/EPS는 표시되지 않음. 투자의견 fallback(priceIndicators)도 같은 TR 계열이라 함께 실패할 수 있음. |

## 1. 투자의견을 가져오지 못하는 원인 (KIS)

### 로그
```
[KIS] opinion: rt_cd=1 msg1=없는 서비스 코드 입니다 code=018260
```

### 분석
- **위치**: `lib/kis-api.ts` — `getInvestmentOpinion(code)` 호출 시 KIS API 응답의 `body.rt_cd`가 `"0"`이 아님.
- **의미**: KIS 측에서 **"없는 서비스 코드 입니다"**를 반환. 즉, 해당 종목코드(018260)에 대해 투자의견 서비스를 제공하지 않거나, 사용 중인 TR ID(`FHKST03110100` / `VFHKST03110100`)가 이 종목을 지원하지 않음.
- **코드 동작**: `body.rt_cd !== "0"`일 때 예외를 던지지 않고 `{ tickerOpinion: null, brokerOpinions: [] }`를 반환하므로, API는 200을 반환하고 화면에는 "증권사별 투자의견이 없습니다" 등 빈 상태로 표시됨.

### 결론
| 구분 | 내용 |
|------|------|
| **원인** | KIS API가 018260에 대해 투자의견 데이터를 제공하지 않음(또는 해당 TR로는 조회 불가). |
| **우리 코드** | 정상. 에러 시 빈 객체 반환 후 200 응답. |
| **대응** | 일부 종목은 KIS 투자의견 미제공이므로, "이 종목은 투자의견 데이터가 없습니다" 같은 안내 문구로 처리하는 것이 좋음. |

## 2. 공시 요약(사업의 내용, MD&A, 주석)을 가져오지 못하는 원인 (DART)

### 로그
```
[DART] getDartDocumentSections: no rcept_no for corp=00126186
```

### 분석
- **위치**: `lib/dart-fundamental.ts` — `getDartDocumentSections(corpCode)` 내부에서 `getDartReportRceptNo(corpCode, key)` 호출.
- **흐름**:
  1. `getDartReportRceptNo`는 DART **list.json**을 호출:  
     `pblntf_detail_ty=A001`(사업보고서), `corp_code=00126186`, `page_no=1`, `page_count=5`.
  2. 여기서 **접수번호(rcept_no)** 를 하나 얻어야 `document.xml` 다운로드 및 00760.xml 파싱으로 이어짐.
  3. `rcept_no`를 못 얻으면 `getDartReportRceptNo`가 `null` 반환 → "no rcept_no for corp=00126186" 로그 발생 → `getDartDocumentSections`는 빈 객체 `{}` 반환.

### rcept_no를 못 얻는 가능 원인
| 가능 원인 | 설명 |
|-----------|------|
| list.json 응답 실패 | HTTP 실패 또는 `status !== "000"` |
| list가 비어 있음 | `data.list`가 빈 배열이거나, A001(사업보고서) 조건에 맞는 건이 없음 |
| 첫 건에 rcept_no 없음 | list[0].rcept_no가 없거나 빈 문자열 |

**corp_code=00126186** 은 삼성에스디에스의 DART 고유번호.  
실제로 DART에 사업보고서가 없거나, 공시 유형/검색 조건(A001만 검색) 때문에 목록에 안 나올 수 있음.

### 결론
| 구분 | 내용 |
|------|------|
| **원인** | `getDartReportRceptNo(00126186)`가 list.json에서 사업보고서(A001) 접수번호를 찾지 못함. |
| **가능 대응** | (1) list.json 응답 로깅(status, list.length, list[0]) 추가하여 원인 확인. (2) A001 없을 때 다른 공시 유형으로 fallback 검색. (3) 해당 corp에 공시가 정말 없으면 "공시 요약 데이터가 없습니다" 안내. |

## 3. DART fnlttSinglAcnt / getDartTrendOnly

### 로그
```
[DART] getDartTrendOnly year=2025 reprt=11014
[DART] getDartTrendOnly year=2024 reprt=11014
[DART] fnlttSinglAcnt corp=00126186 year=2026 status=013 message=조회된 데이타가 없습니다.
```

### 분석
- **year=2026, status=013**: 2026년 데이터는 아직 없음. 정상.
- **year=2025, 2024 reprt=11014**: 3분기(11014)로 조회 시도. 여기서 listLen 등 성공 로그가 없으면, 2025/2024도 해당 reprt로는 데이터가 없을 수 있음. (다른 reprt로는 성공했을 수 있음.)
- 3개년 트렌드가 비어 보이면: 연도별로 11014→11012→11013→11011 fallback이 모두 실패했거나, 일부만 성공해 1~2개년만 채워졌을 수 있음.

### 결론
- 2026 데이터 없음은 정상.
- 2025/2024는 실제 fnlttSinglAcnt 응답(list 유무) 로그를 보면 원인 파악 가능. 필요 시 연도·reprt별 성공/실패 로그 추가 권장.

## 4. 요약 표

| 데이터 | 로그 핵심 | 원인 | 우리 코드 |
|--------|------------|------|-----------|
| **투자의견** | `rt_cd=1 msg1=없는 서비스 코드 입니다 code=018260` | KIS가 018260에 대해 투자의견 미제공 | 정상(빈 데이터 반환) |
| **공시 요약** | `getDartDocumentSections: no rcept_no for corp=00126186` | list.json에서 A001(사업보고서) rcept_no 미조회 | rcept_no 없으면 빈 객체 반환 |
| **DART 3개년** | 2026 데이터 없음, 2025/2024 reprt=11014 시도 | 2026 미제출 정상; 2025/2024는 응답 로그로 추가 확인 필요 | fallback 로직 존재 |

## 5. 권장 조치

1. **투자의견**: KIS가 "없는 서비스 코드"를 주는 종목에 대해, 화면에 "이 종목은 투자의견 데이터를 제공하지 않습니다" 등 명시 안내.
2. **공시 요약**: `getDartReportRceptNo`에서 list.json 응답(status, list 길이, 첫 항목 rcept_no 유무)을 개발 환경에서 로깅해, A001 부재인지/응답 오류인지 구분. 필요 시 다른 공시 유형 fallback 또는 "공시 요약 없음" 안내.
3. **DART 3개년**: 연도·reprt_code별로 fnlttSinglAcnt 성공/실패(또는 listLen) 로그를 추가하면, 연초 등에 데이터가 안 나오는 경우 원인 파악이 쉬움.
