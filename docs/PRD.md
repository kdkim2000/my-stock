# Product Requirements Document (PRD): 국내주식 투자 지원 앱

## 1. Project Overview (프로젝트 개요)

본 프로젝트는 개인 주식 투자자를 위한 **'국내주식 투자 지원 애플리케이션'**입니다.
사용자가 기존에 사용하던 엑셀 데이터를 Google Sheets에 붙여넣기만 하면, 웹/모바일 앱에서 데이터를 실시간으로 동기화하여 **종목별 분석**(종목정보·가치 지표·매매 참고)과 다양한 투자 인사이트(수익률, 승률, 누적 손익 등)를 시각화된 대시보드로 제공합니다. **한국투자증권(KIS) Open API**를 연동하여 보유 종목의 실시간 시세·평가 손익 및 [국내주식] 종목정보·시세분석 데이터를 활용한 가치투자 관점의 참고 정보를 한눈에 확인할 수 있습니다.

## 2. Target Audience (대상 사용자)

* 체계적인 매매 복기와 수익률 관리를 통해 투자 실력을 향상시키고자 하는 **개인 주식 투자자**
* 복잡한 입력 폼 대신 **엑셀/구글 시트(복사&붙여넣기)**를 활용한 직관적인 데이터 입력을 선호하는 사용자

## 3. Core Features (핵심 기능)

### 3.1. Google Sheets 기반 데이터 동기화 (Database)

* 사용자가 구글 시트에 매매 내역을 입력/수정하면 앱에 **즉시 반영 (Read-only 모드 위주)**.
* 앱 내 UI를 통해서도 간단한 **매매 기록 및 복기(저널링)**를 추가하여 구글 시트의 마지막 행(Row)에 **추가 (Append)** 하는 기능 제공.

### 3.2. 실시간 주가 및 포트폴리오 평가 (KIS API 연동)

* **종목명 ↔ 종목코드 매핑 로직 (중요):** 구글 시트에는 'LG전자'와 같이 한글 종목명으로 저장되지만, KIS API는 '066570'과 같은 6자리 코드를 요구하므로 앱 내부에 이를 매핑하는 딕셔너리(Dictionary) 유틸리티가 필수적으로 구현되어야 함.
* **토큰(Token) 캐싱:** KIS API 호출을 위한 Access Token(24시간 유효)을 매번 발급받지 않도록 서버 메모리에 캐싱하는 로직 구현.
* **수익률 자동 계산:** '총 매수 금액', '현재 평가 금액', '실시간 평가 손익'을 자동 계산하여 대시보드 상단에 표시.

### 3.3. 종목별 분석 (종목정보 · 가치 지표 · 매매 가이드)

* **종목별 분석 목록:** 보유·거래 이력이 있는 종목을 평가금액/실현손익 등으로 정렬한 리스트. 종목명 클릭 또는 "상세" 링크로 **종목 상세** 페이지 이동.
* **종목 상세 페이지:** 한 종목에 대해 아래를 한 화면에 표시. 가치투자 관점의 참고 정보만 제공하며, 매수/매도 권유 문구는 사용하지 않음.
  * **시세:** KIS 현재가·전일대비·시고저·거래량·52주 최고/최저.
  * **재무제표:** 대차대조표(자산/부채/자본·유동/비유동), 손익계산서(매출액·영업이익·당기순이익).
  * **재무비율:** 수익성(ROE, ROA, 영업이익률, 순이익률), 안정성(부채비율, 유동비율), 기타(PER, PBR). 데이터 없을 경우 "—" 표시.
  * **투자의견:** KIS 종목 투자의견(의견·목표가·전망) 및 증권사별 투자의견. 참고용이며 투자 권유가 아님.
  * **내 포트폴리오:** 해당 종목 매수/매도 횟수·총 매수/매도 금액·실현손익·실현수익률·승률, 보유 시 수량·평균 단가·평가금액·평가손익.
  * **참고 지표:** 52주 고/저 대비 구간·보유 수익률 등.
  * **AI 분석 및 매매 가이드:** 시세·가치평가·보조지표·내 포트폴리오를 참고로 OpenAI API를 사용해 투자전략 요약과 매매 가이드(참고)를 생성. 참고용이며 매수/매도 권유는 하지 않음. 환경 변수 `OPENAI_API_KEY` 필요.
  * **최근 매매 일지:** 시트 기준 해당 종목 최근 거래 내역.
  * **재무·시세 갱신:** 버튼으로 시세·재무·투자의견 수동 갱신(캐시 스킵) 지원.
* **데이터 소스:** KIS Open API [국내주식] 현재가·일봉 차트(52주 고/저)·종목정보(search-info, PER/PBR 등)·투자의견(invest-opinion); DART 전자공시 API(단일회사 주요계정)로 재무제표·비율. 재무는 KIS 종목정보 우선, 미제공 시 DART 사용. DART 사용 시 환경 변수 `DART_API_KEY`(opendart.fss.or.kr 인증키) 필요.

### 3.4. 인사이트 대시보드 (Data Visualization)

* **핵심 지표 카드 (Summary Cards):** 실현손익, 평가손익(미실현), 전체 승률(참고), 총 자산.
* **차트 (Recharts 활용):** 최근 6개월/1년 누적 수익금 추이, 포지션 집중도(종목별 평가금액 비중), 손익 기여도(보유 종목별 평가손익) 및 손실 포지션 요약.
* **네비게이션:** 대시보드 | 종목별 분석 | 매매일지 | 전략별 성과 구분. 종목별 분석을 진입 포인트로 배치.

### 3.5. 매매 복기 (Trading Journal)

* 개별 거래 내역(Row)을 클릭하면 **상세 내역 모달(Modal)** 또는 사이드 시트 오픈.
* 거래 데이터 원본(날짜, 단가 등)과 함께 **'매매 사유'**, **'감정 상태'**, **'전략 태그'**(#돌파, #눌림목 등)를 기록하고 열람할 수 있는 UI 제공.

## 4. Data Model (데이터 스키마 - Google Sheets 기준)

구글 시트의 첫 번째 행(Header)은 아래와 같은 텍스트 구조를 가집니다. (데이터 매핑 시 참조)

| Google Sheet Header | 데이터 타입 | 설명 및 예시 |
| --- | --- | --- |
| **Date (일자)** | String / Date | 2021-02-17 |
| **Ticker (종목)** | String | GS, LG전자, HL만도 |
| **Type (구분)** | String | 매수 / 매도 |
| **Quantity (수량)** | Number | 50 |
| **Price (단가)** | Number | 40200 |
| **Fee (수수료)** | Number | 72 |
| **Tax (세금)** | Number | 0 |
| **Journal (매매복기)** | Text | (선택) 매매 사유 및 복기 텍스트 |
| **Tags (전략)** | String | (선택) 콤마(,)로 구분된 태그 |

### 4.1. 종목코드 마스터 시트 (선택, 환경 변수 GOOGLE_SHEET_TICKER_MASTER)

동일 스프레드시트 내 별도 탭. 종목명(또는 Ticker) ↔ 6자리 종목코드를 사용자가 시트에서 관리. KIS 현재가 조회 시 1차 소스로 사용.

| 시트 컬럼 | 데이터 타입 | 설명 |
| --- | --- | --- |
| **Ticker** | String | 종목명 또는 식별자 (예: 삼성전자, 005930) |
| **Code** | String | 6자리 종목코드 (예: 005930) |

### 4.2. 종목별 집계 시트 (선택, 환경 변수 GOOGLE_SHEET_AGGREGATION)

동일 스프레드시트 내 별도 탭. 종목별 매수/매도 횟수·금액·실현손익·보유수량 등을 시트에서 확인·편집 가능. Code 컬럼이 있으면 종목코드 마스터가 없을 때 종목코드 fallback으로 사용.

| 시트 컬럼 | 데이터 타입 | 설명 |
| --- | --- | --- |
| **Ticker** | String | 종목 |
| **Code** | String | (선택) 6자리 종목코드 |
| **매수횟수** | Number | 매수 건수 |
| **매도횟수** | Number | 매도 건수 |
| **총매수금액** | Number | 총 매수 금액 |
| **총매도금액** | Number | 총 매도 금액 |
| **실현손익** | Number | 실현 손익 |
| **보유수량** | Number | 현재 보유 수량 |

## 5. UI/UX Requirements

* **반응형 웹 (Responsive Design):** 모바일(아이폰/안드로이드) 환경에서 최우선으로 완벽하게 동작하도록 구성 (Mobile-first 접근).
* **디자인 시스템:** `shadcn/ui`와 `Tailwind CSS`를 활용.
* **테마:** 모던한 다크/라이트 모드 지원. 수익(빨간색/Red)과 손실(파란색/Blue)을 직관적으로 구분할 수 있는 한국 주식시장 색상 테마 적용.

## 6. 제약사항 및 예외 처리 (Constraints & Error Handling)

* **API Rate Limit 방어:** 구글 시트 API와 KIS API의 과도한 호출을 막기 위해 Next.js의 Data Cache 또는 TanStack Query(React Query)를 적절히 활용할 것.
* **빈 값 처리:** 구글 시트에서 수수료, 세금, 매매복기 등이 비어있을 경우(null/undefined) 앱이 크래시 나지 않도록 0 또는 빈 문자열로 안전하게 Fallback 처리할 것.
* **장외 시간 처리:** 주식 시장이 닫힌 주말이나 야간에는 KIS API가 마지막 종가를 정상적으로 반환하도록 에러 핸들링 구성.
* **재무 데이터 연동:** 종목 상세의 재무제표·비율은 KIS 종목정보 API 우선 조회, 미제공 시 DART 전자공시 API로 보완. DART 사용 시 환경 변수 `DART_API_KEY`(opendart.fss.or.kr 인증키)를 설정해야 하며, 미설정 시 해당 종목은 KIS에서 제공하는 항목만 표시된다.

## 7. 배포 및 소스 관리 (Delivery & Operations)

* **소스 관리:** 소스 코드는 **GitHub**에서 버전 관리 및 저장소 호스팅을 수행한다.
* **배포:** 웹 애플리케이션은 **Vercel**에 배포하여 공개하며, Next.js 기반이므로 Vercel과의 연동을 전제로 한다. 프로덕션 환경 변수(Google Sheets, KIS API, DART_API_KEY 등)는 Vercel 프로젝트 설정에서 관리한다.
