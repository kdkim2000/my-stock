# Product Requirements Document (PRD): 국내주식 투자 지원 앱

## 1. 프로젝트 개요

본 프로젝트는 **개인 주식 투자자**를 위한 **국내주식 투자 지원 웹 애플리케이션**입니다.

- 사용자가 **Google Sheets**에 매매 내역을 입력·수정하면, 웹 앱에서 데이터를 동기화하여 **종목별 분석**과 **투자 인사이트**(수익률, 승률, 누적 손익 등)를 시각화된 **대시보드**로 제공합니다.
- **한국투자증권(KIS) Open API**를 연동하여 보유 종목의 실시간 시세·평가 손익 및 종목정보·재무·투자의견·매매동향 등 가치투자 참고 정보를 제공합니다.
- **DART 전자공시 API**로 재무 5개년 트렌드·잠정실적 링크·현금흐름을 보완합니다.
- **OpenAI API**를 사용한 종목별 AI 분석·매매 가이드(참고용)를 제공합니다.
- 가치투자 관점의 **참고 정보만** 제공하며, 매수/매도 권유 문구는 사용하지 않습니다.

---

## 2. 대상 사용자

- 체계적인 매매 복기와 수익률 관리를 통해 투자 실력을 향상시키고자 하는 **개인 주식 투자자**
- **엑셀/구글 시트**(복사·붙여넣기)를 활용한 직관적인 데이터 입력을 선호하는 사용자
- KIS 계좌를 보유하고 실시간 평가 손익·종목 상세 정보를 한곳에서 보고자 하는 사용자

## 3. 핵심 기능

### 3.1. 인증 (Authentication)

- **NextAuth.js** 기반 **Google OAuth 2.0** 로그인.
- 환경 변수: `AUTH_SECRET`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`. 선택적으로 `ALLOWED_EMAIL`로 허용 이메일 제한.
- 로그인 후 **대시보드**(`/dashboard`)로 이동. 비인증 사용자는 로그인 페이지로 리다이렉트.

### 3.2. Google Sheets 기반 데이터 동기화

- 사용자가 구글 시트에 매매 내역을 입력/수정하면 앱에서 **읽기(Read)** 로 반영. 주기 폴링 또는 사용자 새로고침/탭 전환 시 React Query로 최신 데이터 요청.
- 앱 내에서 **매매 기록 및 복기(저널링)** 를 추가할 경우, API Route를 통해 시트 **마지막 행에 Append**.
- 시트 스키마: Date, Ticker, Type(매수/매도), Quantity, Price, Fee, Tax, Journal(매매복기), Tags(전략 태그). 선택 시트: 종목코드 마스터(Ticker, Code), 종목별 집계(매수횟수, 매도횟수, 총매수/매도금액, 실현손익, 보유수량 등).

### 3.3. 실시간 주가 및 포트폴리오 평가 (KIS API)

- **종목명 ↔ 종목코드 매핑**: (1) 종목코드 마스터 시트 (2) 종목별 집계 시트 Code 컬럼 (3) `lib/ticker-mapping.ts` 하드코딩 fallback 순으로 조회. KIS API는 6자리 종목코드를 사용.
- **토큰 캐싱**: Access Token(24시간 유효)을 서버 메모리 및 파일 캐시(`.next/cache/kis-token.json`)로 캐싱하여 재발급 최소화. Vercel 등 서버리스 환경에서는 요청 간 공유를 위해 파일 캐시 사용.
- **수익률 자동 계산**: 총 매수 금액, 현재 평가 금액, 실시간 평가 손익을 API에서 계산해 대시보드·종목 상세에 표시.

### 3.4. 인사이트 대시보드 (Data Visualization)

- **진입점**: `/` → `/dashboard` 리다이렉트. 상단 네비게이션: 대시보드, 종목별 분석(#ticker-analysis), 매매 내역(#journal), 로그아웃.
- **요약 카드(Summary Cards)**: 실현손익, 평가손익(미실현), 전체 승률(참고), 총 자산. 아이콘·로딩 스켈레톤·에러 메시지 포함.
- **종목별 분석**: 보유 종목만 / 과거 투자 포함 전환, 종목 검색, 정렬(종목·평가금액·실현손익·실현수익률·승률·총매수/매도금액). 종목명 클릭 시 종목 상세(`/dashboard/ticker/[id]`) 이동. 보유 뱃지 표시.
- **누적 수익금 추이**: Recharts AreaChart. 6개월/1년 기간 전환. 누적 실현손익 시계열.
- **포지션 집중도**: 종목별 평가금액 비중 가로 막대 차트(상위 N종목 + 기타). 툴팁에 비중·평가금액.
- **손익 기여도 · 손실 포지션**: 종목별 평가손익 가로 막대(수익/손실 색상), 하단 테이블에 손실 포지션(손절·재평가 후보) 링크.
- **전략별 성과(Tags)**: 매매 내역의 Tags(#태그) 기준 집계. 전략별 매도 건수, 실현손익, 승률.
- **매매 내역**: 시트 데이터 테이블. 페이지당 건수 선택, 페이지네이션. 일자·종목·구분·수량·단가·수수료·세금 컬럼.
- **UI/UX**: 반응형, 섹션 카드(rounded-2xl, shadow-sm), 테이블 thead 스타일·빈 상태·에러 상태 통일.

### 3.5. 종목 상세 페이지 (`/dashboard/ticker/[id]`)

- **URL**: `id`는 종목코드(6자리) 또는 종목명. KIS·시트 데이터 조회 시 코드로 정규화.
- **상단 히어로**: 종목명, 종목코드(배지), 현재가, 전일대비(원·%), 갱신 버튼. 보유 시 보유 수량·평가금액·평가손익 요약.
- **섹션 바로가기**: 가로 스크롤 pill 링크(시세, 가치평가, 재무, 비율, 추정실적, 매매동향, 투자의견, DART 손익, 현금흐름, 공시, 포트폴리오, 보조지표, AI 분석, 매매 일지).
- **시세 요약**: 당일 시가·고가·저가·거래량, 52주 최고/최저.
- **가치평가**: PER·PBR(주요 지표 강조), EPS·BPS·Forward EPS. KIS 기준.
- **재무 요약(KIS)**: 대차대조표(자산총계·부채총계·자본총계·유동자산·유동부채), 손익계산서(매출액·영업이익·당기순이익). Recharts 막대 차트로 비교.
- **비율(KIS)**: 재무비율·수익성·안정성·성장성·기타주요비율 카드, 레이더 차트·비율/값 막대 차트.
- **추정실적(KIS)**: 그룹별(추정손익계산서, 투자지표) 키·값·증감율. 증감율 색상(수익/손실).
- **매매동향(KIS)**  
  - **주식현재가 일자별(최근 30일)**: 시가·종가·고가·저가 **선 차트**(4개 라인). API는 최근 30거래일 수신 후 오늘-30일~오늘(일력) 필터 표시.  
  - **투자자별 매매동향(일별)**: 최근 30일(일력) 일자별 API 호출. 개인·기관·외국인 순매수 누적 **라인 차트**.  
  - **일별 매수·매도 체결량**: 최근 30일(일력) 구간. 매수/매도 체결량 **막대 차트**.  
  - **일자 포맷**: 세 차트 모두 **YYYY-MM-DD** 통일(X축·툴팁).
- **투자의견(KIS)**: 종목 투자의견(의견명·목표가·전망), 증권사별 투자의견 테이블(증권사·의견·목표가·제시일). 배지 색상(매수/매도/중립).
- **DART 손익**: 매출액·영업이익·당기순이익 5개년 막대 차트.
- **현금흐름**: 영업·투자·재무 활동 5개년 테이블. DART 데이터.
- **공시**: DART 잠정실적 공시 링크, DART 공시검색 링크.
- **내 포트폴리오**: 해당 종목 매수/매도 횟수·총 매수/매도 금액·실현손익·실현수익률·승률, 보유 수량·평균 단가·평가금액·평가손익.
- **보조지표**: KIS 일봉 기반 RSI(14), MACD(12,26,9). 기준일·과매수/과매도 표시. 참고용.
- **AI 분석 및 매매 가이드**: OpenAI API로 시세·가치·재무·보조지표·내 포트폴리오·최근 매매 일지를 요약해 투자전략·매매 가이드 생성. 참고용, 매수/매도 권유 아님. 환경 변수 `OPENAI_API_KEY`. 분석 요청/다시 분석 버튼, 마크다운 렌더링.
- **최근 매매 일지**: 시트 기준 해당 종목 최근 거래 내역 테이블(일자·구분·수량·단가·금액·비고).
- **재무·시세 갱신**: 버튼으로 KIS·DART·보조지표 수동 갱신(캐시 스킵·refetch).

### 3.6. 매매 복기 (Trading Journal)

- 시트 스키마의 **Journal**(매매복기), **Tags**(전략) 컬럼으로 매매 사유·전략 태그 기록. 앱에서는 시트 Append API로 한 행 추가 시 해당 필드 포함 가능.
- 대시보드 매매 내역 테이블 및 종목 상세 최근 매매 일지에서 Journal/Tags 열람. 상세 모달/사이드 시트는 PRD 범위 내 선택 구현.

## 5. UI/UX 요구사항

- **반응형**: Mobile-first. 모바일·태블릿·데스크탑에서 동작.
- **디자인**: shadcn/ui + Tailwind CSS. 다크/라이트 모드. 수익(빨강)·손실(파랑) 한국 주식 시장 색상 테마(CSS 변수 `--color-profit`, `--color-loss`).
- **대시보드**: 섹션별 카드(rounded-2xl, border, shadow-sm), 아이콘·일관된 thead·빈 상태·에러 메시지. 요약 카드에 아이콘(Banknote, TrendingUp, Target, Wallet).
- **종목 상세**: 히어로(종목명·코드·현재가·갱신)·섹션 네비·섹션별 카드·선/막대 차트·테이블·툴팁 스타일 통일.
- **접근성**: 버튼/링크 aria-label, 시맨틱 헤딩·nav.

## 6. 제약사항 및 예외 처리

- **API Rate Limit**: TanStack Query staleTime/gcTime 및 refetch 간격으로 클라이언트 요청 완화. KIS 1분당 1회 제한(EGW00133) 등은 서버에서 재시도·캐시로 완화.
- **빈 값**: 시트 Fee, Tax, Journal, Tags 등 null/undefined 시 0 또는 "" 로 안전 처리. `lib/normalize-row.ts` 및 타입 optional 활용.
- **장외 시간**: KIS API가 마지막 종가를 반환하는 경우 정상 표시, 에러 시 "장 마감" 등 메시지로 처리·크래시 방지.
- **재무 데이터**: KIS 재무·비율 API 우선. 미제공 시 DART 전자공시 API 보완. `DART_API_KEY` 미설정 시 해당 항목은 KIS만 표시.
- **KIS 토큰**: 서버 메모리·파일 캐시로 24시간 유효 토큰 재사용. Vercel 등에서는 인스턴스 간 공유를 위해 `.next/cache/kis-token.json` 파일 캐시 사용.
