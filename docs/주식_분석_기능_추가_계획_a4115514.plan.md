# 주식 분석 기능 추가 계획

## 1. 활용 가능한 데이터

- **매매 내역** ([types/sheet.ts](types/sheet.ts)): Date, Ticker, Type(매수/매도), Quantity, Price, Fee, Tax, Journal, Tags
- **포트폴리오 요약** ([types/api.ts](types/api.ts)): totalBuyAmount, totalMarketValue, profitLoss, positions(ticker, quantity, buyAmount, marketValue, profitLoss)
- **기존 계산** ([lib/portfolio-summary.ts](lib/portfolio-summary.ts)): 종목별 매수/매도 수량·금액 집계, 보유 비용(buyAmount), KIS 현재가 반영 시 marketValue·profitLoss

위 데이터만으로도 실현손익·수익률·승률·차트용 시계열을 모두 계산 가능합니다.

## 2. 분석 아이디어 (기능 후보)

### 2.1. 실현 손익 / 실현 수익률 (PRD §3.3 지표 카드·차트와 직결)

- **정의**: 이미 **매도**한 부분에 대한 손익.  
  - 매도금액 = Σ(매도 행의 Price×Quantity) − Fee − Tax  
  - 매도원가 = 해당 매도 수량에 대한 매수 비용(평균 단가 또는 FIFO로 추정)  
  - **실현손익 = 매도금액 − 매도원가 − (매도 시 수수료·세금)**  
  - **실현수익률(%) = 실현손익 / 매도원가 × 100**
- **단위**: 전체 합계, **종목별** 합계·수익률
- **활용**: 상단 "총 실현손익" 카드, 종목별 실현 수익률 막대 차트(PRD §3.3), 종목별 분석 테이블

### 2.2. 평가 손익 / 평가 수익률 (이미 부분 구현)

- **정의**: 현재 **보유** 종목의 평가손익(이미 [lib/portfolio-summary.ts](lib/portfolio-summary.ts)와 KIS로 계산됨)
- **추가할 것**: 종목별 **평가수익률(%) = profitLoss / buyAmount × 100** 표시, 대시보드 카드에 "전체 승률" 정의와 연동

### 2.3. 승률 (Win Rate, PRD §3.3)

- **정의 (제안)**:  
  - **거래 1건** = 동일 종목에서 "매수 → 매도"가 완료된 단위.  
  - 단순화: **매도 건수** 기준으로, 해당 매도가 실현손익 ≥ 0이면 승, &lt; 0이면 패.  
  - **승률(%) = 승리한 매도 건수 / 전체 매도 건수 × 100**
- **추가 옵션**: 종목별 승률, 전략(Tags)별 승률
- **활용**: Summary 카드 "전체 승률", Donut/Pie(PRD §3.3)

### 2.4. 종목별 분석 요약

- **종목당 한 줄**:  
  Ticker | 매수횟수 | 매도횟수 | 총매수금액 | 총매도금액 | 실현손익 | 실현수익률(%) | 보유수량 | 평가손익 | 평가수익률(%) | 승률(%)
- **데이터 소스**: 매매 내역 집계 + 기존 포트폴리오 positions
- **UI**: 대시보드 섹션 "종목별 분석" 테이블, 정렬/필터(수익률 순, 실현손익 순 등)

### 2.5. 누적 수익금 추이 (PRD §3.3)

- **정의**: 일자별로 "그날까지의 실현손익 누적" + "(선택) 당일 기준 평가손익"을 더한 **누적 수익금 시계열**
- **계산**:  
  - 매매 내역을 날짜 순 정렬 후, 매도 건마다 실현손익을 계산해 해당 일자에 합산.  
  - 일자별로 누적합 → 차트용 데이터 포인트 (date, cumulativeRealized, cumulativeTotal 등)
- **기간 필터**: 최근 6개월 / 1년 (PRD §3.3)
- **UI**: Line Chart 또는 Area Chart (Recharts)

### 2.6. 매수/매도 비중 및 승률 차트 (PRD §3.3)

- **매수/매도 비중**: 전체 거래 금액(또는 건수) 중 매수 합계 vs 매도 합계 비율 → Donut/Pie
- **승률 시각화**: 승 vs 패 건수 비율 → Donut/Pie
- **UI**: Recharts PieChart/Donut

### 2.7. 전략·태그(Tags)별 분석 (선택)

- **정의**: Journal/Tags 필드에서 #돌파, #눌림목 등 태그를 파싱해, **태그별** 실현손익·수익률·승률 집계
- **전제**: 시트에 Tags가 일관되게 입력되어 있을 때 유의미
- **UI**: "전략별 성과" 테이블 또는 소형 차트

### 2.8. 매매 복기(Journal) 연동 (PRD §3.4와 연계)

- 거래 행 클릭 시 상세 모달에서 Journal/Tags 편집은 기존 계획 유지
- **분석 연계**: 태그별 분석(2.7)은 이 데이터를 읽기만 하면 됨

## 3. 데이터·계산 레이어 제안

- **신규 모듈**  
  - `lib/analysis.ts` (또는 `lib/realized-pnl.ts`):  
    - 매매 내역만으로 **실현손익·실현수익률** 계산(매도 원가 추정: 평균 단가 또는 FIFO).  
    - **승률** 계산(매도 건 단위).  
    - **일자별 누적 수익금** 시계열 생성.  
    - **종목별 집계** (실현손익, 실현수익률, 매수/매도 횟수, 승률).
  - 필요 시 `lib/analysis-tags.ts`: Tags 파싱 및 전략별 집계(2.7).
- **API**  
  - `GET /api/analysis/summary`:  
    - 전체 실현손익, 전체 승률, (선택) 종목별 요약 배열.  
  - `GET /api/analysis/cumulative-pnl?period=6m|1y`:  
    - 누적 수익금 추이 시계열.  
  - 기존 `GET /api/kis/portfolio-summary`는 그대로 두고, 프론트에서 "실현"과 "평가"를 함께 표시.
- **타입**  
  - [types/api.ts](types/api.ts)에 분석 응답 타입 추가:  
    - `AnalysisSummaryResponse`, `CumulativePnlPoint`, `TickerAnalysisRow` 등.

## 6. 정리

- **분석의 핵심**은 "실현손익/실현수익률"과 "승률" 정의를 하나로 정한 뒤, `lib/analysis.ts`에서 매매 내역만으로 계산하는 것.
- **종목별·일자별·태그별** 분석은 동일 집계 로직을 단위(종목/일자/태그)만 바꿔 재사용.
- **PRD §3.3**의 차트 3종(누적 수익금, 종목별 실현 수익률, 매수/매도·승률)과 상단 지표를 이 단계대로 채우면, "다양한 분석"의 1차 목표를 충족할 수 있습니다.

이 계획을 기준으로 Phase 1부터 순서대로 구현하면 됩니다.
