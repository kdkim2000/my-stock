# 상세페이지 하이브리드 아키텍처 리팩토링

## 현재 구조 요약

- **데이터 소스**: [app/api/kis/stock-info/route.ts](app/api/kis/stock-info/route.ts)에서 KIS 현재가·일봉·종목정보(PER/PBR) 조회 후, **DART로 PER/PBR/EPS 보정**([getFinancialsByStockCode](lib/dart-api.ts)). [app/api/fundamental/valuation/route.ts](app/api/fundamental/valuation/route.ts)에서도 KIS + DART 혼합으로 PER/PBR/ROE/EV EBITDA 계산.
- **클라이언트**: [TickerDetailContent.tsx](components/dashboard/TickerDetailContent.tsx)가 `stock-info`, `fundamental/valuation`, `fundamental/financials`, `dart/financials`, `kis/opinion` 등을 **각각 useQuery로 분리 호출**.
- **DART**: [getFundamentalFinancials](lib/dart-fundamental.ts)는 **11011(사업보고서)만** 사용. [getDartReportRceptNo](lib/dart-fundamental.ts)는 list.json으로 **A001(사업보고서)**만 검색.

---

## 1단계: API 병렬 호출 라우트 구성

**목표**: 한 번의 요청으로 Source A(KIS)와 Source B(DART)를 `Promise.all`로 병렬 호출하는 **단일 진입점** 제공.

- **신규 라우트**: `app/api/fundamental/route.ts` (기존 `valuation`·`financials`와 역할 구분되므로 신규 생성 권장)
  - **GET** `?code=066570` (필수 6자리)
  - **Source A (KIS)** — `Promise.all([ getKisPriceAndIndicators(code), getInvestmentOpinion(code) ])`
    - `getKisPriceAndIndicators`: 현재가 시세 + PER/PBR/EPS/BPS. **FHKST01010400**은 KIS 포털에 명시된 TR이 없어, 우선 **기존 조합 유지**: `getPriceInfo(code)`(FHKST01010100) + `getKisStockFundamentals(code, currentPrice)`(FHKST02010100 search-info). 응답에 `per`, `pbr`, `eps`, `bps` 및 있으면 `forward_eps` 등 추정치 필드 추출.
    - `getInvestmentOpinion(code)`: 기존 [app/api/kis/opinion/route.ts](app/api/kis/opinion/route.ts) 호출 또는 `lib/kis-api.ts`의 `getInvestmentOpinion` 직접 사용.
  - **Source B (DART)** — `Promise.all([ getDartTrendOnly(code), getDartPreliminaryAndDocument(corpCode) ])`
    - `getDartTrendOnly`: 아래 3단계에서 정의. 최근 3개년 트렌드용 multiYear + 분기 Fallback.
    - `getDartPreliminaryAndDocument`: list.json으로 최근 2개월 잠정실적 공시 링크 조회 + 기존 `getDartDocumentSections`(사업보고서 기반 MD&A 등).
  - 응답 타입: `{ kis: { priceInfo, per, pbr, eps, bps, forwardEps?, opinion } }, dart: { multiYear, preliminaryLink?, document } }` 형태로 통일.
- **클라이언트**: [TickerDetailContent.tsx](components/dashboard/TickerDetailContent.tsx)와 [useFundamentalData.ts](hooks/useFundamentalData.ts) 수정
  - 상세 페이지는 `**/api/fundamental?code=...` 한 번만** 호출하거나, 기존처럼 여러 useQuery를 쓰되 **데이터 소스 책임**은 서버의 단일 “fundamental” 라우트에 몰지 않고, **1) KIS 지표/투자의견**, **2) DART 트렌드/공시** 두 가지로 명확히 나누어 호출할지 선택 필요.  
  - 권장: `**/api/fundamental` 하나에서 Promise.all로 A+B 수행**하고, 클라이언트는 해당 API 1회 호출로 상단(KIS)·중단(DART 차트)·하단(DART 링크·아코디언)에 필요한 데이터를 모두 받음.

## 2단계: 최신 지표는 KIS 전면 위임 (수동 계산 제거)

- **제거할 로직**:
  - [app/api/kis/stock-info/route.ts](app/api/kis/stock-info/route.ts): 71~84행 부근의 **DART fallback** 전체 삭제. (`getFinancialsByStockCode` 호출 및 `per`/`pbr`/`eps`/`bps`를 DART로 채우는 부분.)
  - [app/api/fundamental/valuation/route.ts](app/api/fundamental/valuation/route.ts): DART로 PER/PBR/EPS 계산·보정하는 부분(53~59행), ROE/EV EBITDA는 DART `dartFundamental?.multiYear?.[0]` 등으로 계산하는 부분 유지할지 여부.  
    - 요구사항상 “DART는 과거 트렌드·텍스트 전용”이므로, **valuation 라우트는 KIS만 사용**하거나, **valuation 라우트 자체를 폐지**하고 `/api/fundamental` 응답의 KIS 블록만 사용하도록 할 수 있음.
- **KIS 값만 사용**:
  - PER, PBR, EPS, BPS는 **오직** `getKisStockFundamentals`(search-info FHKST02010100) 및 현재가 기반 보정 결과만 사용.  
  - KIS 응답에 **추정치(Forward EPS 등)** 필드가 있으면 파싱해 `forwardEps` 등으로 노출.
- **정리**: `lib/dart-api.ts`의 `getFinancialsByStockCode`를 stock-info/valuation에서 **더 이상 호출하지 않도록** 제거. DART는 아래 3단계용 함수만 사용.

## 3단계: DART는 과거 트렌드·텍스트 전용으로 축소

- **트렌드 전용 함수** (신규 또는 [lib/dart-fundamental.ts](lib/dart-fundamental.ts) 확장):
  - **목적**: “최근 3개년 재무제표 트렌드 차트”용 **multiYear**만 생성. 사업보고서(11011)만 쓰지 않고, **최신 연도부터 11014(3Q) → 11012(2Q) → 11013(1Q) → 11011(연간)** 순으로 찔러서 **데이터가 존재하는 가장 최신 reprt_code**를 사용하는 Fallback.
  - [getFnlttSinglAcnt](lib/dart-api.ts): 이미 `reprtCode` 파라미터 지원. 호출부에서 연도별로 `["11014","11012","11013","11011"]` 순으로 시도.
  - 연도는 **현재 연도 ~ (현재-2)** 까지 3개년. 각 연도당 위 reprt_code 순으로 한 번만 성공하면 해당 연도 데이터로 push.
- **잠정실적 링크** (DART list.json):
  - [list.json](https://opendart.fss.or.kr/api/list.json) 호출 시 `corp_code`, `bgn_de`/`end_de`(최근 2개월), 공시유형으로 **잠정실적**에 해당하는 항목 검색. (DART 공시유형 코드는 포털/문서 기준으로 지정. 예: A002, A003 등 또는 키워드 "잠정".)
  - 조회된 목록에서 `rcept_no` 등으로 **공시 보기 URL** 생성 (예: `https://dart.fss.or.kr/dsaf001/main.do?rcpNo={rcept_no}`)하여 `preliminaryLink`로 반환.
- **기존 document(MD&A)**:
  - `getDartDocumentSections`는 그대로 두고, **사업보고서 기반** MD&A·사업의 내용·주석만 하단 아코디언용으로 사용. 잠정실적은 “링크”만 제공.

## 4단계: UI 컴포넌트 레이아웃 재배치

- **상단 헤드라인**: 
  - KIS에서 받은 **실시간 PER, PBR, EPS**(및 선택적 BPS·Forward EPS)를 **크게** 배치.
  - 뱃지: **"최근 실적 반영(KIS 기준)"** 문구 표시.
  - 현재가·전일대비 등 기존 시세 블록 유지.
- **중단 차트**:
  - DART `multiYear`만 사용해 **매출액·영업이익**(·당기순이익) **막대 그래프** 유지.  
  - 대차대조표/손익계산서/재무비율 등 **단일 연도 DART 테이블**은 제거하거나 “참고용” 접이식으로 축소할지 결정. (요구사항은 “트렌드 차트”만 명시.)
- **하단 텍스트**:
  - **DART 잠정실적**: 최근 2개월 잠정실적 공시가 있으면 **링크 버튼** 노출 (새 탭에서 DART 페이지 열기).
  - **사업보고서 요약**: 기존 “공시 요약” 아코디언 유지 — 사업의 내용, 이사의 경영진단(MD&A), 주석.
- **제거·이동**:
  - 가치평가 섹션에서 **ROE/EV EBITDA**는 DART 의존이므로, KIS 전용으로 바꾼 뒤에는 **제거**하거나, DART 트렌드의 “최신 연도” 한 건으로만 계산해 “참고” 표시할 수 있음. (요구사항은 KIS 위임·DART 축소이므로 제거 권장.)
  - `/api/dart/financials` 호출은 **상세페이지에서 제거**하고, 단일 `/api/fundamental`(또는 KIS 전용 + DART 전용 2개) 응답만 사용.

## 구현 순서 제안

1. **lib/dart-fundamental.ts** (및 필요 시 dart-api.ts): 분기 Fallback(11014→11012→11013→11011) 적용한 **getDartTrendOnly** 구현; list.json 기반 **잠정실적 링크** 조회 함수 추가.
2. **app/api/fundamental/route.ts** 신규: Source A(KIS)와 Source B(DART) `Promise.all` 구성, 응답 스키마 정의.
3. **app/api/kis/stock-info/route.ts**: DART fallback 제거. PER/PBR/EPS/BPS는 KIS만 사용.
4. **app/api/fundamental/valuation/route.ts**: DART 기반 PER/PBR/EPS 계산 제거 또는 라우트 폐지 후 `/api/fundamental`로 통합.
5. **hooks/useFundamentalData.ts** (또는 신규 훅): `/api/fundamental` 1회 호출로 전환.
6. **TickerDetailContent.tsx**: 데이터 소스를 새 API에 맞춘 뒤, 상단(KIS 지표+뱃지) → 중단(DART 차트) → 하단(잠정실적 링크+아코디언) 순으로 레이아웃 재배치; 불필요 섹션(단일 연도 DART 재무 테이블 등) 제거 또는 축소.

## 주의사항

- **FHKST01010400**: 사용자 요청에는 “현재가 시세 FHKST01010400”이 있으나, 공개 문서에서 해당 TR 확인되지 않음. 구현 시 **기존 FHKST01010100(현재가) + FHKST02010100(종목정보)** 조합을 유지하고, 추후 KIS 포털에서 01010400 확인 시 교체하는 방식**을 권장.
- **DART 잠정실적**: list.json의 공시유형 코드(잠정실적 해당)는 DART API 문서 또는 개발가이드로 확인 필요.
- **캐시**: `/api/fundamental`에 적절한 Cache-Control(revalidate 300~600초) 및 필요 시 `revalidate=1` 쿼리 지원.
